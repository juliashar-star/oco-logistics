# Glossary — словарь простыми словами

Чтобы не теряться в терминах. Если встретили незнакомое слово — ищите здесь или попросите
в Cursor: «объясни простыми словами, что такое …».

- **MVP** — самая первая, минимальная версия продукта, которой уже можно пользоваться.
- **Фронтенд** — то, что видит пользователь в браузере (экраны, кнопки).
- **Бэкенд** — серверная часть, которую пользователь не видит (логика, работа с базой).
- **API** — способ, которым программы «разговаривают» друг с другом. Мы через API общаемся с APIShip.
- **Агрегатор (APIShip)** — посредник, дающий доступ ко многим службам доставки через один API.
- **База данных (БД)** — место, где хранятся все данные (заказы, клиенты, статистика).
- **PostgreSQL** — конкретная база данных, которую мы используем.
- **ORM / Prisma** — удобный «переводчик» между кодом и базой; через него пишем и читаем данные.
- **Миграция** — изменение структуры базы (добавили таблицу/поле). Prisma делает это аккуратно.
- **Docker / контейнер** — «коробка», в которой приложение запускается одинаково и на компьютере,
  и на сервере. Поэтому переезд на сервер безболезненный.
- **Docker Compose** — запуск нескольких таких коробок (приложение + база) одной командой.
- **Репозиторий / git** — «история версий» вашего кода; можно откатиться назад, если что-то сломали.
- **Деплой** — выкладка готового продукта на сервер, чтобы им могли пользоваться.
- **VPS / сервер** — «компьютер в дата-центре», работающий круглосуточно. У нас — в России.
- **root** — главный администратор сервера (полный доступ).
- **`.env`** — файл с секретами и настройками (ключи, пароли). В код и в git не попадает.
- **Sandbox (песочница)** — тестовый режим, где можно пробовать без реальных последствий и денег.
- **HTTPS / SSL** — защищённое соединение (замочек в браузере). Обязательно, где есть личные данные.
- **ПВЗ** — пункт выдачи заказов.
- **SLA** — обещанный уровень сервиса (например, срок доставки).
- **Carrier Score** — наш собственный рейтинг качества перевозчика (0–100) по реальным данным.
- **ПДн (персональные данные)** — данные, по которым можно узнать человека (ФИО, телефон, адрес).
- **152-ФЗ** — закон РФ о персональных данных; требует хранить ПДн россиян на серверах в РФ.
- **Оператор ПДн** — тот, кто обрабатывает персональные данные; регистрируется в Роскомнадзоре.
- **ОКВЭД** — код вида деятельности бизнеса; для IT-услуг нужны свои коды.
- **Decision Engine** — наш «движок выбора» доставки: правила «Быстро/Дёшево/Оптимально».
- **Эндпоинт (endpoint)** — конкретный «адрес» запроса в API.
